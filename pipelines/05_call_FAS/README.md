# Module 05: Call FAS (FIRE Area Score)

This module implements **FAS (FIRE Area Score)**, a quantitative metric developed in our study to measure chromatin accessibility enrichment within centromere dip regions (CDRs).

Rather than relying solely on average FIRE coverage, FAS integrates both the **magnitude** and **genomic extent** of FIRE enrichment, providing a robust quantitative measurement of centromeric chromatin accessibility across different cell types and experimental conditions.

The pipeline accepts Fiber-seq FIRE coverage files (`all_element_coverages.bed.gz`) generated by the Fiber-seq FIRE pipeline together with CDR and active HOR annotations. It automatically extracts active HOR regions and their flanking sequences, calculates FAS scores, performs quality control, and generates publication-ready visualizations.

---

## Method Overview

The workflow consists of four major steps.

### 1. Extract active HOR regions

FIRE coverage records overlapping active HOR regions and their flanking sequences are efficiently extracted from bgzip-compressed coverage files using **tabix**.

### 2. Construct accessibility profiles

FIRE coverage is aggregated into fixed genomic bins (default: **5 kb**) to generate continuous chromatin accessibility profiles across each CDR and its surrounding background.

### 3. Estimate background accessibility

Background FIRE coverage is estimated from regions outside the CDR after robust Winsorization, reducing the influence of local outliers and sequencing coverage fluctuations.

### 4. Calculate FAS

For each CDR, the pipeline estimates a robust background FIRE coverage from the surrounding non-CDR regions and quantifies the cumulative FIRE enrichment across the entire CDR relative to this background.

The pipeline reports:

- Raw FAS
- log₂-transformed FAS
- Background FIRE coverage
- CDR FIRE coverage
- FIRE enrichment
- QC figures

## Workflow

```text
Fiber-seq FIRE coverage (.bed.gz)
        + active HOR annotation
                │
                ▼
        tabix region extraction
                │
                ▼
      Fixed-bin FIRE profile
                │
                ▼
 Background FIRE estimation
      (Robust Winsorization)
                │
                ▼
         FAS calculation
                │
                ▼
      QC plots & summary tables
```

## Usage

```bash
bash call_FAS_pipeline_V1.0.sh \
    -fa fire_manifest.tsv \
    -cen active_hor_cdr.bed \
    -o output_directory
```

### Optional parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `--bin_size` | 5000 | Bin size (bp) |
| `--smooth_k` | 10 | Smoothing window size |
| `--cdr_expansion` | 0 | CDR expansion size (bp) |
| `--out_CDR_cutoff` | 0.10 | Winsorization cutoff outside the CDR |
| `--in_CDR_cutoff` | 0.05 | Winsorization cutoff inside the CDR |

## Output

For each sample, the pipeline generates:

- Processed FIRE profiles
- CDR accessibility profiles
- Background FIRE estimates
- FAS summary tables
